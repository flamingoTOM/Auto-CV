# Auto-CV 简历生成助手

## 技能说明

本技能通过对话式问答，引导用户逐步填写简历信息。
每执行完一步，自动更新 `resume.tex` 文件，编译渲染出简历PDF。
工作内容描述采用 **STAR 原则**（Situation/Task → Action → Result）进行优化。

---

## 执行流程

开始时向用户说明：
> "您好！我是晓煜开发的简历机器人，我将分 7 个步骤引导您完成简历，现在开始"

逐步完成以下各步骤，**每步收集完毕后立即更新 `resume.tex`，再进入下一步**。

---

### 第零步：环境检查（自动，无需用户操作）

在进入第一步之前，先静默检查 XeLaTeX 是否可用：

```bash
xelatex --version
```

**若命令成功**：直接进入第一步，无需提示用户。

**若命令失败（未安装）**：
1. 告知用户：「检测到您尚未安装 LaTeX 编译环境，正在为您自动安装 MiKTeX，请稍候……」
2. 执行安装（使用 Windows 内置的 winget）：
   ```bash
   winget install --id MiKTeX.MiKTeX -e --accept-source-agreements --accept-package-agreements
   ```
3. 安装完成后，刷新环境变量并再次验证：
   ```bash
   refreshenv 2>/dev/null || true
   xelatex --version
   ```
4. **若验证成功**：告知用户「MiKTeX 安装完成！」，进入第一步。
5. **若仍然失败**：提示用户手动安装：
   > 「自动安装未能生效（可能需要重启终端）。请访问 https://miktex.org/download 手动下载安装，安装完成后重新运行 `/Auto-CV`。」
   然后**终止本次执行**。

> 注意：winget 在 Windows 10 1709+ 及 Windows 11 上默认可用。若 winget 本身不存在，改为提示手动安装。

---

### 第一步：个人信息

依次询问：
1. **姓名**（必要）
2. **籍贯**（必要，如陕西省西安市） 
3. **邮箱**（必要）
4. **电话**（必要）
5. **政治面貌**（非必要，如：中共党员、共青团员、群众）
6. **通讯地址**（非必要）
7. **个人主页**（非必要，如GitHub、简书等）
8. **作品集链接**（非必要）

更新 `resume.tex` 中 `\MyName{}` 及各 `\SimpleEntry{\textbf{xxx：}xxx}` 字段。

---

### 第二步：教育背景

询问用户有几段教育经历，然后逐一询问：
- **学校名称**（必要）
- **专业**（必要）
- **学历**（必要，如：本科、硕士、博士）
- **时间段**（必要）

更新 `\section{教育背景}` 下的 `\datedsubsection` 内容。

---

### 第三步：照片（可选）

询问用户是否需要添加照片：
- **需要**：提示将照片命名为 `you.jpg` 放入 `/images` 文件夹，保留 `\yourphoto{0.14}`
- **不需要**：在该行前加 `%` 注释掉

> %0.15的意思是，照片的宽度是页面宽度的0.15倍，调整大小，避免遮挡文字

---

### 第四步：实习经历

询问用户有几段实习经历，然后逐一询问：
- **公司名称**（必要）
- **岗位**（必要）
- **实习时间**（必要，格式：YYYY.M - YYYY.M）
- **工作内容**（必要，用户自然语言描述）

处理方式：
1. 将工作内容精炼为 **20-30 字**，写入 `{工作内容：xxx}`
2. 用 STAR 原则扩展为 **2-3 点**，每点 **30-60 字**，配上一些可以量化的数据，写入 `\Content` 下方各 `{ 1.xxx}` `{ 2.xxx}` `{ 3.xxx}`
3. 时间倒叙排列，最新的实习经历排在最前面

输出格式：
```latex
\datedsubsection{\textbf{公司名称（实习）}，岗位}{开始 - 结束}
{工作内容：精炼后20-30字描述}
\Content
{ STAR第一点，30-60字}
{ STAR第二点，30-60字}
{ STAR第三点，30-60字（如有）}
```

---

### 第五步：项目经历

询问用户有几段项目经历，然后逐一询问：
- **项目名称**（必要）
- **奖项**（非必要，如有则附在标题后）
- **项目时间**（必要，格式：YYYY.M - YYYY.M）
- **工作内容**（必要，用户自然语言描述）

处理方式：
1. 将工作内容精炼为 **20-30 字**，写入 `{工作内容：xxx}`
2. 用 STAR 原则扩展为 **2-3 点**，每点 **30-60 字**，配上一些可以量化的数据，写入 `\Content` 下方
3. 时间倒叙排列，最新的项目经历排在最前面

输出格式：
```latex
\datedsubsection{\textbf{项目名称}，奖项（如有）}{开始 - 结束}
{工作内容：精炼后20-30字描述}
\Content
{ STAR第一点，30-60字}
{ STAR第二点，30-60字}
{ STAR第三点，30-60字（如有）}
```

---

### 第六步：学生工作

询问用户有几段学生工作，**建议填写 1-3 段**，然后逐一询问：
- **工作部门**（必要）
- **工作岗位**（必要）
- **工作内容**（必要）：
  - 必须包含：`负责xxxxx工作`
  - 如有成果可附加：`，成果发表在xxxx`

输出格式：
```latex
\datedsubsection{\textbf{部门}，岗位，\textit{负责xxx工作}}{开始 - 结束}
```
或（有成果时）：
```latex
\datedsubsection{\textbf{部门}，岗位，\textit{负责xxx工作，成果发表在xxx}}{开始 - 结束}
```

---

### 第七步：个人评价

询问用户以下信息：
- **英文水平**（如：CET-6 xxx分、雅思 x 分、托福 xxx 分等）
- **技术栈**（编程语言、工具、框架等）
- **爱好特长**（可选）
- **其他补充**（可自由拓展）

综合以上信息，写 **3-5 点**个人评价，每点 **30-45 字**，适度自由发挥，更新 `\section{个人评价}` 下的内容。

输出格式（每点用 `\SimpleEntry` 或直接段落）：
```latex
\section{个人评价}
{1.xxx}

{2.xxx}

{3.xxx}
```

---

## 完成后

1. 展示更新后 `resume.tex` 的关键部分供用户确认
2. 自动执行编译：
   ```bash
   bash build.sh
   ```
   - **编译成功**：告知用户「✓ resume.pdf 已生成，可直接打开查看！」
   - **编译失败**：展示错误信息，提示用户检查 LaTeX 语法或联系开发者
3. 询问用户是否需要调整任何部分
